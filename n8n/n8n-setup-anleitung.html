1
n8n lokal starten (Docker)

Voraussetzung: Docker Desktop installiert

Docker Desktop für Windows herunterladen und installieren (kostenlos).
Nach Installation: Terminal öffnen und prüfen:

docker --version
docker compose version

n8n starten

1
In den Projektordner navigieren: e:\Kita connect\n8n\
2
Terminal öffnen und ausführen:
docker compose up -d
3
Browser öffnen: http://localhost:5678
4
Login: Benutzer admin / Passwort kita-connect-local
✓ n8n ist einsatzbereit. Daten werden persistent in einem Docker Volume gespeichert.

Nützliche Docker-Befehle

# n8n stoppen
docker compose down

# Logs anzeigen
docker compose logs -f n8n

# n8n neustarten
docker compose restart n8n
2
Credentials in n8n einrichten

In n8n: Settings → Credentials → New

Folgende Credentials anlegen — jeweils mit dem angegebenen Namen (so referenzieren die Workflows sie):

1. Supabase Postgres

Name in n8n:Supabase Postgres (Kita Connect)
Typ:Postgres
Host:db.DEIN-PROJECT-ID.supabase.co
Port:5432
Database:postgres
User:postgres
Password:→ Supabase Dashboard → Settings → Database → Password
SSL:Aktivieren (require)

2. Resend SMTP

Name in n8n:Resend SMTP (Kita Connect)
Typ:SMTP
Host:smtp.resend.com
Port:465 (SSL)
User:resend
Password:→ Resend Dashboard → API Keys → New Key

3. Telegram Bot

Name in n8n:Telegram Bot (Kita Connect)
Typ:Telegram API
Access Token:→ @BotFather in Telegram → /newbot → Token kopieren
Wichtig: Die Credentials werden verschlüsselt in n8n gespeichert (Encryption Key im docker-compose.yml). Den Key vor dem produktiven Einsatz ändern und sicher aufbewahren.
3
Telegram Broadcast-Kanal einrichten

Schritt-für-Schritt

1
In Telegram: Neuen Kanal erstellen → Kita Connect Aktuell → als öffentlichen Kanal anlegen mit Handle z.B. @kitaconnect
2
In Telegram: @BotFather öffnen → /newbot → Name: Kita Connect Bot → Token kopieren
3
Bot als Administrator zum Kanal hinzufügen (Kanal-Infos → Admins → Bot hinzufügen)
4
Channel-ID ermitteln: Browser öffnen → https://api.telegram.org/bot[TOKEN]/getUpdates → eine Testnachricht im Kanal schicken → chat.id aus der Antwort kopieren (meist -100xxxxxxxxxx)
5
In n8n: Environment Variable TELEGRAM_CHANNEL_ID setzen (Settings → Variables) mit der Channel-ID
4
Supabase Datenbank vorbereiten

SQL ausführen

1
Supabase Dashboard öffnen → dein Projekt → SQL Editor
2
Datei n8n/supabase-setup.sql öffnen → Inhalt kopieren → in SQL Editor einfügen → Run
3
Prüfen ob die Tabellen notifications und broadcasts unter Table Editor erscheinen
✓ Webhook-URLs zeigen direkt auf eugen-m.n8n.irn.hk — Supabase kann sie ohne Tunnel direkt erreichen.

Workflows testen (ohne Supabase-Trigger)

Workflow öffnen → Test Workflow → Webhook-Node → Listen for Test Event → dann curl:

# Beispiel: Ticket-Webhook testen
curl -X POST https://eugen-m.n8n.irn.hk/webhook-test/ticket-created \
  -H "Content-Type: application/json" \
  -d '{
    "event": "ticket.created",
    "ticket_id": "test-123",
    "parent_id": "DEINE-USER-ID",
    "subject": "Testticket",
    "body": "Das ist ein Test.",
    "created_at": "2025-05-15T10:00:00Z"
  }'
5
Workflows importieren

JSON-Workflows in n8n laden

1
eugen-m.n8n.irn.hk öffnen → Workflows → oben rechts ⋯ → Import from File
2
Nacheinander importieren aus dem Ordner n8n/workflows/:
01-ticket-created.json
02-broadcast.json
03-parent-registered.json
04-observation-created.json
3
Jeden Workflow öffnen → Credentials zuweisen (die vorher angelegten) → Aktivieren
✓ Alle Workflows sind jetzt aktiv und warten auf Webhook-Aufrufe.
6
Environment Variables in n8n

Settings → Variables → Neue Variable anlegen

TELEGRAM_CHANNEL_IDChannel-ID des Telegram-Kanals (z.B. -100123456789)
VAPID_PUSH_ENDPOINTURL deiner Next.js Push-API Route (z.B. https://kita-connect.app/api/push/send)
VAPID_PRIVATE_KEYVAPID Private Key für Web Push (generiert in Next.js-App)
7
Später: Migration zu n8n Cloud

Warum n8n Cloud?

n8n ist ein deutsches Unternehmen — DSGVO-konform, EU-Hosting, keine Daten in die USA. Für Produktion empfohlen (~20 €/Monat Starter).

Migration in 3 Schritten

1
Alle Workflows exportieren: Workflow → ⋯ → Export (JSON)
2
n8n Cloud Account anlegen → Workflows importieren → Credentials neu anlegen
3
In supabase-setup.sql: alle host.docker.internal:5678 durch deine n8n Cloud Webhook-URLs ersetzen → SQL erneut ausführen
✓ Supabase Webhooks können dann direkt n8n Cloud erreichen — kein Tunnel mehr nötig.
